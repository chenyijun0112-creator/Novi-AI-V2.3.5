首页推荐视频改版 Home

页面定位

首页推荐 轻推荐 + 快速决策 + 导流进入创作。
灵感流 沉浸浏览 + 连续种草 + 内容消费。
说明 首页推荐作为快速决策入口，旨在帮助用户更高效地完成模板选择与创作转化；灵感流作为沉浸式内容消费场，承担更完整的刷内容、互动与种草职责。首页负责“选”，灵感流负责“刷”，两者不是重复建设，而是前后衔接的互补关系。

一、需求概述

首页推荐区域由旧长视频老模板展示改为 Trending 推荐热门趋势视频及 Clip 短视频双 Tab 展示。
模板视频预览页优化，界面与灵感流视频样式统一，支持点赞、分享、创建同款。
模板展示由固定 `3:4` 改为瀑布流交错展示（`3:4` 或 `4:3`）。

三、功能需求

1. Tab切换：推荐区 Tab 提供 `Trending` 与 `Clip` 两个池，用户点击后在当前页面内完成切换。
2. 卡片展示：首页卡片仅展示静态封面、播放按钮与模板标题；模板区采用双列弱瀑布流布局，左右等宽，仅使用 `3:4` 与 `4:3` 两种固定卡片比例，不允许自由高度。卡片按固定节奏交错排布，既保留一定内容流感，也保证首页整齐度与可扫视性。
卡片比例映射规则：
原始视频素材比例不直接决定首页卡片比例。 竖屏素材（宽高比 < 1）映射到 `3:4`。 横屏素材（宽高比 > 1）映射到 `4:3`。 方形素材（宽高比 = 1）默认映射到 `3:4`。 首页封面/首帧按 `cover` 填充容器，允许裁切；点击进入预览页后，再按真实视频比例播放。
3. Tab吸顶：Tab 区在页面滚动后吸顶至系统状态栏下方，保证用户在浏览不同模板时可随时切换推荐池，无需回到顶部重新操作。
4. 预览承接：点击任一卡片后进入单条模板预览页，页面自动播放真实模板视频，并展示账号、点赞、分享与 `Create Same` 主按钮。
5. 生成链路：`Create Same` 进入生成页时，需带入模板对应的基础信息；从生成页返回时，应回到原模板预览页，保证链路闭环。

四、推荐排序规则

多重优先级：
Trending：点赞数 > 后台排序权重 > 发布时间 Clip：发布时间 > 点赞数 > 后台排序权重
排序说明：
点赞数，按从高到低排序。 后台排序权重，按从高到低排序（数字越高，权重越高）。 发布时间，按从新到旧排序。
Trending：优先承载高热度、高验证度模板，用于提升用户快速决策效率。
Clip：优先承载新品分类模板，用于补充新鲜感与冷启动曝光，同时保留热度与后台调控能力。

Inspiration 灵感广场

一、需求概述

一句话说明 将底部 `Clips Tab` 升级为 `Inspiration` 灵感广场，以全屏竖版短视频流承载官方模板内容，建立从内容消费到模板创建的高转化发现链路。
背景问题 当前 `Clips` 承载的特效模板内容老旧，页面形态偏模板列表，不符合海外用户主流的短视频浏览习惯，导致内容停留与创作转化能力有限。
版本目标 通过“先刷视频种草，再一键创建同款”的路径，提升灵感广场内容消费深度、模板点击率与进入生成页后的最终生成转化。

二、需求目标与范围

目标用户：海外内容消费型用户、创意探索型用户、被模板内容吸引并具备潜在生成意愿的轻中度创作者。
本期目标：验证灵感流是否能提升模板“被看见”与“被点击创建”的能力，优先关注发现链路，不扩展复杂社区能力。
本期包含：Inspiration 信息流、官方模板内容、点赞、分享、Create Same、文生/图生目标页带参预填。
本期不含：UGC 发布、评论、关注、个人主页、复杂推荐算法、多频道分类、点赞列表。

三、前台功能需求

1. Tab 改版：底部导航原 `Clips` 入口更名为 `Inspiration`，维持原导航层级与位置，降低用户学习成本。
2. 页面形态：首页采用全屏竖版视频流，单屏展示单条模板视频，支持上下切换内容。
3. 内容策略：V1 仅展示官方模板内容，统一以 `NoviAI_Official` 身份露出，强化内容可信度与平台官方调性。
4. 内容信息层：单条内容需展示官方账号、官方标识、模板标题、点赞数、分享数与主行动按钮 `Create Same`。
5. 模板类型：同时支持文生视频与图生视频模板；图生视频模板需具备参考图信息，并可传递至目标功能页。
6. 分享能力：支持复制链接、下载视频；分享对象为当前单条模板视频，链接需可回流至对应模板内容。
7. 分享预览页：分享链接落地后需提供模板视频预览页，用于承接外部渠道点击流量；页面需展示 `Novi AI` 品牌头、模板视频预览、官方账号 `NoviAI_Video`、模板标题与主行动按钮 `Create Same →`。用户在该 H5 页点击 `Create Same →` 时：已安装 App 则唤起并回流至对应单个模板详情页；未安装则跳转应用下载页（App Store / Google Play）。
8. 推荐排序：前台默认优先展示未看过视频；同等条件下，点赞高内容优先于默认排序内容；内容刷到底后循环继续。

四、交互逻辑说明

1. 进入页面：用户进入 `Inspiration` 后默认落在推荐视频流，视频自动播放、单条循环，不再展示传统模板列表说明。
2. 播放控制：用户点击视频区域可暂停/继续播放；页面底部展示播放进度条，用于反馈当前视频播放进度。
3. 上下切换：支持上下切换下一条/上一条模板视频；切换后自动播放新视频；内容刷到底后继续循环，不出现空态拦截。
4. 点赞逻辑：游客点击点赞跳转登录页；登录用户可完成点赞/取消点赞，不展示成功 Toast，但按钮状态与推荐信号需同步生效。
5. 分享逻辑：
1）用户点击分享按钮后，页面展示底部分享弹窗与背景蒙层；支持 `Copy Link`、`Download Video` 两项操作。 2）`Copy Link` 实际复制的并非单独 URL，而是一整段带深链的分享文案，便于用户直接转发至外部渠道。 3）接收方点击分享文案中的链接后，若设备已安装 Novi AI App，则优先唤起 App 并回流至对应模板详情/模板内容页；若未安装 App，则可直接跳转 H5 承接页。 4）H5 分享预览页需默认展示模板视频首帧与播放按钮，顶部仅露出 `Novi AI` 品牌名，底部保留 `NoviAI_Video`、模板标题与 `Create Same →` 主按钮，减少冗余说明文案，聚焦回流转化。 5）用户在 H5 承接页点击 `Create Same →`：若设备已安装 Novi AI App，则直接唤起 App，并回流至当前分享对应的单个模板详情页（与深链模板 ID 一致）；若未安装 App，则直接跳转应用下载页（App Store / Google Play）。 6）用户点击关闭按钮或蒙层外区域时，分享弹窗与背景蒙层需同时消失，不保留残留遮罩。
示例分享文案：Check out this amazing AI video: https://novi.ai/inspiration/sample-office-bunny Tap the link or copy this message to open it in Novi AI and view the full content.
6. 下载逻辑：点击下载后分享弹窗消失，保留背景蒙层并展示“下载中”弹窗；下载完成后关闭弹窗，并展示 `Saved to Album` Toast。
7. Create Same：按钮位于底部主行动区。游客无需登录即可进入功能页，缩短从“被种草”到“尝试生成”的路径。
8. 带参预填：文生视频模板跳转 `Text to Video` 并带入模型、Prompt、比例、分辨率、时长；图生视频模板跳转 `Image to Video` 并额外带入参考图。

五、后台需求

1. Novi AI-AI中台-素材中台，新增灵感流管理界面
字段： 分类：固定归属为“灵感流”，用于区分该内容与其他模板分类。 模板类型：支持文生视频 / 图生视频 / Agent视频，用于区分模板内容形态及前台跳转逻辑。 名称：用于后台内容识别的名称字段，可作为运营管理、内容检索及配置区分的基础标识。 使用次数：记录模板被“Create Same”调用的累计次数，用于衡量模板转化表现。 点赞次数：记录模板累计点赞总数，口径为各端点赞数据汇总。 平台：支持 iOS / Android / Web 等，用于控制模板在哪些端展示或生效。 排序：排序权重字段，数值越大优先级越高，用于控制灵感流内容展示顺序。 分享URL：模板视频对应的分享链接，用于复制分享、外部回流及分享预览页承接。 状态：启用 / 禁用，用于控制模板是否在灵感流中生效展示。 创建时间：记录模板内容创建时间，用于后台管理与内容追踪。 更新时间：记录模板配置最近一次修改时间，用于版本追踪与运营核对。 操作： 编辑：支持修改模板基础信息、展示平台、排序权重、分享URL及状态配置。 删除：支持移除对应灵感流模板配置，删除后不再参与前台展示。
2. 分享链路支持：支持“复制模板视频分享URL + 回流 + 分享预览页”。

商店评价引导

1. 需求概述 新增商店评分评价引导弹窗，用于在不打断用户核心创作链路的前提下，提升满意用户的应用商店评分转化，并收集低分用户的具体反馈，作为后续产品优化依据。
2. 流程说明 1）当用户触发评分引导场景后，页面展示背景蒙层与五星评分弹窗。 2）若用户点击 `4-5` 星，则不增加额外确认步骤，直接跳转对应应用商店评分页。 3）若用户点击 `1-3` 星，则不跳转应用商店，直接进入低分反馈问卷弹窗。 4）若用户完成低分反馈提交，则页面展示感谢反馈 Toast。 5）若用户点击右上角 `X`，则关闭当前弹窗并结束本次引导。
3. 五星评分弹窗触发条件 1）用户首次导出视频成功后触发，仅触发一次。 2）用户点击个人中心 `Review Us` 入口时触发。
4. 交互逻辑 五星评分弹窗： 1）用户点击 `1-3` 星后，当前评分弹窗关闭，并立即进入低分反馈问卷弹窗。 2）用户点击 `4-5` 星后，当前弹窗关闭，并直接跳转对应应用商店评分页。 3）用户点击右上角 `X` 后，关闭评分弹窗，不再继续后续流程。 问卷收集弹窗： 1）用户可多选不满意原因，选项包括 `Video quality is poor`、`The result didn't match my idea`、`Characters or scenes are wrong`、`Generation is too slow`、`Unsatisfied with the interaction experience`、`Other`。 2）当用户选择 `Other` 时，可进一步输入补充反馈内容。 3）`Submit` 按钮默认置灰；当用户至少选择一个原因，或填写 `Other` 文本后，按钮变为可点击。 4）用户点击 `Submit` 后，关闭问卷弹窗，并展示感谢反馈 Toast：`Thanks for your feedback. We'll keep improving Novi AI for you.` 5）用户点击右上角 `X` 后，关闭问卷弹窗，不再继续反馈提交流程。
